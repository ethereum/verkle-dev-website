IPA Multipoint

Introduction​

This document explains how to open multiple polynomials at multiple different points. Ultimately, we use one IPA proof, 1 commitment and 1 scalar. This is the batched setting.

Statement​

Given mmm IPA commitments C0=[f0(X)]...Cm−1=[fm−1(X)]C_0 = [f_0(X)] ... C_{m-1} = [f_{m-1}(X)]C0​=[f0​(X)]...Cm−1​=[fm−1​(X)], prove evaluations:

f0(z0)=y0⋮fm−1(zm−1)=ym−1f_0(z_0) = y_0 \\\vdots \\f_{m-1}(z_{m-1}) = y_{m-1}f0​(z0​)=y0​⋮fm−1​(zm−1​)=ym−1​

where zi∈{0,...,d−1}z_i \in \{0,...,d-1\}zi​∈{0,...,d−1}

Proof​

  1. Let r←H(C0,...Cm−1,z0,...,zm−1,y0,...,ym−1)r \leftarrow H(C_0,...C_{m-1}, z_0, ..., z_{m-1}, y_0, ..., y_{m-1})r←H(C0​,...Cm−1​,z0​,...,zm−1​,y0​,...,ym−1​)
g(X)=r0f0(X)−y0X−z0+r1f1(X)−y1X−z1+…+rm−1fm−1(X)−ym−1X−zm−1g(X) = r^0 \frac{f_0(X) - y_0}{X-z_0} + r^1 \frac{f_1(X) - y_1}{X-z_1} + \ldots +r^{m-1} \frac{f_{m-1}(X) - y_{m-1}}{X-z_{m-1}}g(X)=r0X−z0​f0​(X)−y0​​+r1X−z1​f1​(X)−y1​​+…+rm−1X−zm−1​fm−1​(X)−ym−1​​

The prover starts off by committing to g(X)g(X)g(X), we denote this by DDD or [g(X)][g(X)][g(X)].

The provers job is to now convince the verifier that DDD is a commitment to g(X)g(X)g(X). We do this by evaluating g(X)g(X)g(X) at some random point ttt

We split the evaluation into two parts g1(t)g_1(t)g1​(t) and g2(t)g_2(t)g2​(t), g2(t)g_2(t)g2​(t) can be computed by the verifier, while g1(t)g_1(t)g1​(t) cannot, because it involves random evaluations at the polynomials fi(X)f_i(X)fi​(X).

  • The verifier is able to compute the g2(t)g_2(t)g2​(t).
  • The prover will compute g1(t)g_1(t)g1​(t) and send a proof of it's correctness.
g1(t)=rifi(t)t−zig_1(t) = r^i \frac{f_i(t)}{t-z_i}g1​(t)=rit−zi​fi​(t)​
g2(t)=riyit−zig_2(t) = r^i \frac{y_i}{t-z_i}g2​(t)=rit−zi​yi​​
  1. Let t←H(r,D)t \leftarrow H(r,D)t←H(r,D)

We note that g1(X)=rifi(X)X−zig_1(X) = r^i \frac{f_i(X)}{X-z_i}g1​(X)=riX−zi​fi​(X)​, however, we specify it as rifi(X)t−zir^i \frac{f_i(X)}{t-z_i}rit−zi​fi​(X)​ because the latter is also able to prove an opening for g1(t)g_1(t)g1​(t) and the verifier is able to compute the commitment for it.

Now we form two IPA proofs:

  • One for g1(X)g_1(X)g1​(X) at ttt. We call this π\piπ
  • One for g(X)g(X)g(X) at ttt. We call this ρ\rhoρ

The prover now computes y=g1(t)y = g_1(t)y=g1​(t)

The proof consists of D,(π,y),ρD, (\pi, y), \rhoD,(π,y),ρ

In this non-aggregated variation, the prover does not need to add [g1(X)][g_1(X)][g1​(X)] to the transcript.

Verification​

The Verifier ultimately wants to verify that DDD is the commitment to the polynomial g(x)g(x)g(x).

The verifier computes rrr and ttt.

The verifier also computes g2(t)g_2(t)g2​(t), we mentioned above that they can do this by themselves.

Computing g(t)g(t)g(t)​

The verifier now needs to compute g(t)g(t)g(t):

g(t)=g1(t)−g2(t)g(t) = g_1(t) - g_2(t)g(t)=g1​(t)−g2​(t)

  • We know that g1(t)g_1(t)g1​(t) was supplied in the proof as yyy.
  • g2(t)g_2(t)g2​(t) can be computed by the verifier.

Hence the verifier can compute g(t)g(t)g(t).

Note however, that they cannot be sure that g1(t)g_1(t)g1​(t) is the correct computation by the prover. They need to build [g1(X)][g_1(X)][g1​(X)] themselves and verify it against g1(t)g_1(t)g1​(t)

Computing [g1(X)][g_1(X)][g1​(X)]​

This is g1(X)g_1(X)g1​(X):

g1(X)=rifi(X)t−zig_1(X) = r^i \frac{f_i(X)}{t-z_i}g1​(X)=rit−zi​fi​(X)​

Hence [g1(X)][g_1(X)][g1​(X)] is:

[g1(X)]=rit−ziCi[g_1(X)] = \frac{r_i}{t-z_i}C_i[g1​(X)]=t−zi​ri​​Ci​

The verifier is able to compute this themselves, and so is able to verify that g1(t)g_1(t)g1​(t) was computed correctly using IPA_VERIFY.

We can now call IPA_VERIFY using

  • [g1(X)][g_1(X)][g1​(X)]
  • g1(t)g_1(t)g1​(t)
  • π\piπ

Is g(t)g(t)g(t) correct?​

Note now that since g1(t)g_1(t)g1​(t) was verified to be correct and g2(t)g_2(t)g2​(t) was computed by the verifier, we can be sure that g(t)g(t)g(t) is correct.

Verify g(x)g(x)g(x) at ttt​

We now call IPA_VERIFY using:

  • D=[g(X)]D = [g(X)]D=[g(X)] *
  • g(t)g(t)g(t)
  • ρ\rhoρ

*In actuality, it's not DDD but an augmented DDD, but this works at a higher level and does not ruin the explanation.

Complexity​

The communication complexity of this protocol is two IPA proofs, 1 scalar and 1 commitment. We can get a better protocol by aggregating things together!

Aggregation​

We now present a protocol to aggregate the two IPA proofs together, only requiring one IPA proof.

Prover​

  1. Let q←H(t,[g1(X)])q \leftarrow H(t, [g_1(X)])q←H(t,[g1​(X)])

The prover no longer computes an IPA proof for g1(X)g_1(X)g1​(X) and g(X)g(X)g(X) instead they combine them using qqq.

g3(X)=g1(X)+q⋅g(X)g_3(X) = g_1(X) + q \cdot g(X)g3​(X)=g1​(X)+q⋅g(X)

Now we form an IPA Proof for g3(X)g_3(X)g3​(X) at ttt. Lets call this σ\sigmaσ.

The prover still computes y=g1(t)y = g_1(t)y=g1​(t)

The proof consists of D,σ,yD, \sigma, yD,σ,y

Verifier​

In the previous step, the verifier called [g1(X)][g_1(X)][g1​(X)] ,g1(t)g_1(t)g1​(t) with π\piπ, we delay this verification and instead compute:

  • [g3(X)]=[g1(X)]+q⋅[g(X)][g_3(X)] = [g_1(X)] + q \cdot [g(X)][g3​(X)]=[g1​(X)]+q⋅[g(X)]
  • g3(t)=g1(t)+q⋅g(t)g_3(t) = g_1(t) + q \cdot g(t)g3​(t)=g1​(t)+q⋅g(t)

We now call IPA_VERIFY using:

  • [g3(X)][g_3(X)][g3​(X)]
  • g3(t)g_3(t)g3​(t)
  • σ\sigmaσ

With overwhelming probability over qqq this will only return true iff [g1(X)][g_1(X)][g1​(X)] and [g(X)][g(X)][g(X)] opened at ttt are g1(t)g_1(t)g1​(t) and g(t)g(t)g(t) respectively, from the equation.

Complexity​

The communication complexity of the protocol is 1 IPA Proof, 1 commitment and 1 scalar.

Do we need to add [g1(X)][g_1(X)][g1​(X)] to the transcript?​

In the KZG document, this is h(X)h(X)h(X)

If we were able to avoid this, then we could save a lot on prover time, as they could evaluate each fif_ifi​ at ttt, then do ri⋅fi(t)t−zi\frac{r^i \cdot f_i(t)}{t - z_i}t−zi​ri⋅fi​(t)​ instead of needing to first compute ri⋅fi(X)t−zi\frac{r^i \cdot f_i(X)}{t - z_i}t−zi​ri⋅fi​(X)​. (In the non-batched version)

However, we do need to do this because the challenge qqq is aggregating [g1(X)][g_1(X)][g1​(X)] and [g(X)][g(X)][g(X)], we need to bind [g1(X)][g_1(X)][g1​(X)] to the challenge qqq.

I generate a completely random variable ttt.

If I want to add together f(X,t)f(X, t)f(X,t) and g(x)g(x)g(x)
